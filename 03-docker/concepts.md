# 🐳 What is Docker?
**Docker is a containerization platform** that allows you package an applications together with all its dependencies into a single unit called a container, so it runs the same way everywhere.

> 📦 “A lightweight box that contains your app + everything it needs to run.”

### why we use containers?
- Package an application with everything it needs
- Run it the same way everywhere
- Deploy it quickly and reliably
- Required for Kubernetes
- Perfect for CI/CD pipelines

### How deployments might happen with and without containers?

| Without Containers | With Containers   |
| ------------------ | ----------------- |
| Manual setup       | Image-based setup |
| Environment drift  | Same everywhere   |
| Slow scaling       | Fast scaling      |
| Hard rollback      | Easy rollback     |
| Server-dependent   | Portable          |
